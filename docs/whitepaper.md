# KOOSY: AI 시대 1인 비즈니스 인프라

> 서버 없이, 팀 없이, 월 4,000원으로 운영하는 콘텐츠 비즈니스 시스템
> **+ 디지털 자아 구축 파이프라인**

---

## Executive Summary

KOOSY는 1인 콘텐츠 크리에이터와 소규모 자영업자를 위한 **서버리스 풀스택 인프라**입니다. GitHub를 중심으로 모든 데이터를 관리하고, AI(Claude Code)와 협업하여 개발부터 운영까지 혼자서 처리할 수 있는 시스템입니다.

**단순한 콘텐츠 도구가 아닙니다.**

일상의 모든 경험과 대화를 캡처하고, 구조화된 데이터로 축적하여, 궁극적으로 **나만의 LLM**을 만들 수 있는 **디지털 자아 구축 시스템**입니다.

**핵심 가치:**
- 서버 비용 0원 (GitHub Pages + GitHub Actions)
- 월 운영비 4,000원 (GitHub Pro, Private 레포)
- 콘텐츠 생산 속도 10배 향상
- 데이터 축적 → RAG → Fine-tuning → 자아 복제

## 3. 핵심 경쟁력: 생산 속도

### 워크플로우 비교

**기존 방식:**
```
경험 → 메모 정리 → PPT 제작 → 촬영 → 편집 → 업로드
                    ↓
              며칠 ~ 일주일
```

**KOOSY 방식:**
```
경험 → Parksy Capture → 웹페이지 자동 생성 → 화면녹화 → 업로드
                    ↓
                수십 분
```

### 단계별 속도 차이

| 단계 | 기존 | KOOSY |
|------|------|-------|
| 자료 정리 | 수동 메모 | Parksy Capture 자동 |
| 시각 자료 | PPT 수동 제작 | 웹페이지 자동 생성 |
| 촬영 | 카메라 세팅 | 화면녹화 바로 시작 |
| 편집 | 수 시간 | **없음** |
| 다국어 | 자막 별도 제작 | 크롬 번역 자동 |
| 도구 필요 | 구매/학습 | Claude로 즉시 개발 |

### 노편집 콘텐츠 파이프라인

- 태블릿 화면녹화 + 실시간 오버레이
- 와콤 스타일 판서 (손가락/펜 스트로크 구분)
- 상황판 오버레이 (현재 위치/진행 상황)
- **후보정 없이 바로 업로드**

**결과:** 남들이 1개 만들 때 10개 만들 수 있음

## 4. 장기 게임: 디지털 자아 구축

### 데이터 파이프라인

```
일상의 모든 경험/대화
        ↓
  Parksy Capture (안드로이드 공유 기능)
        ↓
  GitHub 레포 (마크다운 자동 축적)
        ↓
    ┌───┴───┐
    ↓       ↓
   RAG    Fine-tuning
    ↓       ↓
지식 검색   나만의 LLM
    ↓       ↓
    └───┬───┘
        ↓
   "나" 복제 가능
```

### 3단계 게임

| 단계 | 목표 | 결과 |
|------|------|------|
| **단기** | 콘텐츠 빠르게 생산 | YouTube 수익 |
| **중기** | 지식 베이스 축적 | RAG 검색 가능 |
| **장기** | Fine-tuning | 나만의 LLM, 자아 복제 |

### 왜 이게 중요한가?

1. **휘발되지 않음**: 모든 경험이 구조화된 데이터로 남음
2. **검색 가능**: RAG로 과거의 나에게 질문 가능
3. **학습 가능**: Fine-tuning으로 나처럼 생각하는 AI
4. **복제 가능**: 멀티태스킹, 스케일 가능
5. **양도 가능**: Exit 시 데이터와 모델 가져갈 수 있음

## 6. 비즈니스 모델

### 운영 구조

```
본사 (dtslib.com)
│
├── Collaborator로 멤버 등록
├── 인프라 제공 (6개 레포 세팅)
├── AI 개발 지원
├── 데이터 파이프라인 구축
│
└── Exit 시: 누적 수익의 30%로 독립
    - 모든 레포 Fork
    - 축적된 데이터 가져감
    - 독립 개발 가이드 제공
```

### 왜 턴키가 아닌 Collaborator 모델인가?

**실제 테스트 결과:**
- 비개발자에게 턴키로 전달 시도 → **실패**
- GitHub, Termux, Claude Code 체인 중 하나만 막혀도 끝
- 가르쳐서 혼자 하게 하는 건 **불가능**

**결론:** Collaborator로 같이 키우고, Exit 시 30%로 독립

| 턴키 모델 | Collaborator 모델 |
|-----------|-------------------|
| 가르쳐서 혼자 하게 함 | 같이 키움 |
| 기술 허들 높음 | 기술 허들 없음 |
| 유지보수 본인 책임 | 본사가 관리 |
| 데이터 축적 안 됨 | 데이터 같이 쌓임 |
| 실패 확률 높음 | 성공 확률 높음 |

## 7. AI 시대 적합성

### 패러다임 변화

**과거:**
```
회사 = 정규직 팀 + 사무실 + 협업 도구 + 고정비
```

**현재 (AI 시대):**
```
1인 + Claude = 풀스택 개발
필요시 용역 계약
고정비 최소화
데이터 축적 → 자기만의 AI
```

### KOOSY가 이 패러다임에 맞는 이유

1. **GitHub 중심** - 협업 도구 불필요
2. **Claude Code 통합** - AI가 개발 파트너
3. **정적 사이트** - 서버 관리 불필요
4. **자동화 파이프라인** - GitHub Actions
5. **데이터 축적** - RAG/Fine-tuning 준비
