# dtslib.com Mobile-first AI Builder

> 스마트폰 하나로 1인 IT 기업을 구축하는 보일러플레이트 시스템
> **첫 번째 구현: KOOSY (콘텐츠 비즈니스)**

---

## Executive Summary

**dtslib.com Mobile-first AI Builder**는 비개발자가 스마트폰만으로 완전한 IT 인프라를 구축할 수 있게 해주는 **보일러플레이트 시스템**입니다.

GitHub 6개 레포 네트워크 + Claude Code + Termux 조합으로, PC 없이도 풀스택 개발부터 앱 배포까지 가능합니다.

**KOOSY**는 이 시스템의 첫 번째 브랜치 구현으로, 1인 콘텐츠 크리에이터를 위한 서버리스 인프라입니다.

### 핵심 가치

| 레벨 | 가치 | 결과 |
|------|------|------|
| **인프라** | 서버 0원, 월 4,000원 운영 | 비용 장벽 제거 |
| **속도** | 콘텐츠 생산 10배 빠름 | 경쟁 우위 |
| **축적** | 모든 경험 → 구조화 데이터 | 디지털 자아 |
| **확장** | 브랜치 복제 → 계열사 | 홀딩 컴퍼니 |

## 1. 제품 구조

### 1.1 보일러플레이트 시스템

```
dtslib.com Mobile-first AI Builder
│
├── 코어 인프라
│   ├── 6개 레포 네트워크 템플릿
│   ├── Staff Portal 프레임워크
│   ├── PWA/APK 빌드 파이프라인
│   └── 데이터 캡처 시스템
│
├── 브랜치 구현
│   ├── KOOSY (콘텐츠 비즈니스) ← 첫 번째
│   ├── [다음 브랜치] ← 복제 가능
│   └── [계열사 N] ← 무한 확장
│
└── AI 레이어
    ├── Claude Code 통합
    ├── RAG 지식 검색
    └── Fine-tuning 파이프라인
```

### 1.2 왜 "보일러플레이트"인가?

**복제 가능한 구조:**
- `config.json` 수정만으로 새 브랜치 생성
- 6개 레포 Fork → 즉시 독립 운영
- 동일한 인프라, 다른 비즈니스

**브랜치 예시:**
| 브랜치 | 타겟 | 콘텐츠 |
|--------|------|--------|
| KOOSY | 중년층 | 경험 공유 유튜브 |
| [미정] | MZ세대 | 취미 커뮤니티 |
| [미정] | B2B | 컨설팅 포털 |

## 2. 핵심 경쟁력: 생산 속도 10배

### 2.1 워크플로우 비교

**기존 방식:**
```
경험 → 메모 정리 → PPT 제작 → 촬영 → 편집 → 업로드
                    ↓
              며칠 ~ 일주일
```

**dtslib Builder 방식:**
```
경험 → Parksy Capture → 웹페이지 자동 생성 → 화면녹화 → 업로드
                    ↓
                수십 분
```

### 2.2 단계별 속도 차이

| 단계 | 기존 | dtslib Builder |
|------|------|----------------|
| 자료 정리 | 수동 메모 | Parksy Capture 자동 |
| 시각 자료 | PPT 수동 제작 | 웹페이지 자동 생성 |
| 촬영 | 카메라 세팅 | 화면녹화 바로 시작 |
| 편집 | **수 시간** | **없음** |
| 다국어 | 자막 별도 제작 | 크롬 번역 자동 |
| 도구 필요 | 구매/학습 | Claude로 즉시 개발 |

### 2.3 노편집 콘텐츠 파이프라인

- 태블릿 화면녹화 + 실시간 오버레이
- 와콤 스타일 판서 (손가락/펜 구분)
- 상황판 오버레이 (현재 위치/진행 상황)
- **후보정 없이 바로 업로드**

**결과:** 남들이 1개 만들 때 10개 만들 수 있음

## 3. 장기 게임: 디지털 자아 구축

### 3.1 데이터 파이프라인

```
일상의 모든 경험/대화
        ↓
  Parksy Capture (안드로이드 공유 기능)
        ↓
  GitHub 레포 (마크다운 자동 축적)
        ↓
    ┌───┴───┐
    ↓       ↓
   RAG    Fine-tuning
    ↓       ↓
지식 검색   나만의 LLM
    ↓       ↓
    └───┬───┘
        ↓
   "나" 복제 가능
```

### 3.2 3단계 게임

| 단계 | 목표 | 결과 |
|------|------|------|
| **단기** | 콘텐츠 빠르게 생산 | YouTube 수익 |
| **중기** | 지식 베이스 축적 | RAG 검색 가능 |
| **장기** | Fine-tuning | 나만의 LLM, 자아 복제 |

### 3.3 왜 이게 중요한가?

1. **휘발되지 않음**: 모든 경험이 구조화된 데이터로 남음
2. **검색 가능**: RAG로 과거의 나에게 질문 가능
3. **학습 가능**: Fine-tuning으로 나처럼 생각하는 AI
4. **복제 가능**: 멀티태스킹, 스케일 가능
5. **양도 가능**: Exit 시 데이터와 모델 가져갈 수 있음

## 4. 시스템 아키텍처

### 4.1 6개 레포지토리 네트워크

| 레포 | 용도 | 호스팅 |
|------|------|--------|
| **main-branch** | 랜딩페이지 + Staff Portal | GitHub Pages |
| **posts** | HTML 블로그 | GitHub Pages |
| **logs** | Parksy Capture 데이터 | GitHub API |
| **media** | 이미지/음원 | jsDelivr CDN |
| **pwa** | 프로그레시브 웹앱 | Vercel |
| **apk-lab** | 안드로이드 APK | GitHub Actions |

### 4.2 서버리스 아키텍처

**기존 방식:**
```
사용자 → 서버 → 데이터베이스 → 응답
         ↓
    월 수만원 비용
    서버 관리 필요
```

**dtslib Builder 방식:**
```
사용자 → GitHub Pages (정적) → 즉시 응답
         ↓
    비용 0원
    관리 불필요
    GitHub SLA 보장
```

### 4.3 모바일 퍼스트 개발

- **Termux + Claude Code** = 스마트폰에서 풀스택 개발
- **Parksy Capture** = 어디서든 데이터 캡처
- **화면녹화** = 태블릿에서 바로 콘텐츠 제작
- **PC 없이 모든 작업 가능**

## 6. 비즈니스 모델

### 6.1 Collaborator 모델

```
본사 (dtslib.com)
│
├── Collaborator로 멤버 등록
├── 인프라 제공 (6개 레포 세팅)
├── AI 개발 지원
├── 데이터 파이프라인 구축
│
└── Exit 시: 누적 수익의 30%로 독립
    - 모든 레포 Fork
    - 축적된 데이터 가져감
    - 독립 개발 가이드 제공
```

### 6.2 왜 턴키가 아닌 Collaborator인가?

**실제 테스트 결과:**
- 비개발자에게 턴키로 전달 시도 → **실패**
- GitHub, Termux, Claude Code 체인 중 하나만 막혀도 끝
- 가르쳐서 혼자 하게 하는 건 **불가능**

**결론:** Collaborator로 같이 키우고, Exit 시 30%로 독립

| 턴키 모델 | Collaborator 모델 |
|-----------|-------------------|
| 가르쳐서 혼자 하게 함 | 같이 키움 |
| 기술 허들 높음 | 기술 허들 없음 |
| 유지보수 본인 책임 | 본사가 관리 |
| 실패 확률 높음 | 성공 확률 높음 |

## 8. 최종 비전

### 8.1 개인 IT 역량의 진화

```
취미 레벨 → dtslib Builder → 1인 IT 기업
                ↓
        브랜치 복제 → 계열사 네트워크
                ↓
        서버 LLM → 온톨로지 → 자아 복제
```

### 8.2 AI 시대 패러다임

**과거:**
```
회사 = 정규직 팀 + 사무실 + 협업 도구 + 고정비
```

**현재 (AI 시대):**
```
1인 + Claude = 풀스택 개발
필요시 용역 계약
고정비 최소화
데이터 축적 → 자기만의 AI
```

### 8.3 dtslib.com의 위치

dtslib.com은 단순한 도구가 아닙니다.

**취미 수준의 개인 IT 역량**을
**완전한 1인 IT 기업 수준**으로 끌어올리는
**Mobile-first AI Builder** 플랫폼입니다.

KOOSY는 그 첫 번째 증명입니다.

```
Staff Portal
├── HQ 브리핑 (공지/튜토리얼)

|------|----------|

```
Parksy Capture → GitHub logs 레포 → 마크다운 축적
                                          ↓
HTML 포스터 → GitHub posts 레포 → GitHub Pages 배포
                                          ↓
화면녹화 → YouTube 업로드 → 수익화
                                          ↓
전체 데이터 → RAG/Fine-tuning → 나만의 LLM
