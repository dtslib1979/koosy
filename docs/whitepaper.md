# KOOSY: AI 시대 1인 비즈니스 인프라

> 서버 없이, 팀 없이, 월 4000원으로 운영하는 콘텐츠 비즈니스 시스템

---

## Executive Summary

KOOSY는 1인 콘텐츠 크리에이터와 소규모 자영업자를 위한 **서버리스 풀스택 인프라**입니다. GitHub를 중심으로 모든 데이터를 관리하고, AI(Claude Code)와 협업하여 개발부터 운영까지 혼자서 처리할 수 있는 시스템입니다.

**핵심 가치:**
- 서버 비용 0원 (GitHub Pages + GitHub Actions)
- 월 운영비 4,000원 (GitHub Pro)
- 정규직 채용 없이 AI + 용역으로 운영
- 편집 없이 콘텐츠 양산

## 3. 기술적 차별점

### 3.1 서버리스 아키텍처

**기존 방식:**
```
사용자 → 서버 → 데이터베이스 → 응답
         ↓
    월 수만원 비용
    서버 관리 필요
    장애 대응 필요
```

**KOOSY 방식:**
```
사용자 → GitHub Pages (정적) → 즉시 응답
         ↓
    비용 0원
    관리 불필요
    GitHub SLA 보장
```

### 3.2 AI 협업 개발

| 항목 | 기존 | KOOSY |
|------|------|-------|
| 개발 | 개발자 채용 (월 300만원+) | Claude Code (API 비용만) |
| 디자인 | 디자이너 채용 | Claude + Canva |
| DevOps | DevOps 엔지니어 | GitHub Actions (무료) |
| 유지보수 | 상시 인력 | 필요시 AI 호출 |

### 3.3 노편집 콘텐츠 파이프라인

**기존 영상 제작:**
```
촬영 → 편집 (수 시간) → 인코딩 → 업로드
```

**KOOSY 방식:**
```
화면녹화 + 실시간 오버레이 → 바로 업로드
│
├── 와콤 스타일 판서 (손가락/펜 구분)
├── 상황판 오버레이
└── 후보정 불필요
```

## 4. 비즈니스 모델

### 운영 구조

```
본사 (dtslib.com)
│
├── Collaborator로 멤버 등록
├── 인프라 제공 (6개 레포 세팅)
├── AI 개발 지원
│
└── Exit 시: 누적 수익의 30%로 독립
```

### 왜 턴키가 아닌 Collaborator 모델인가?

| 턴키 모델 | Collaborator 모델 |
|-----------|-------------------|
| 가르쳐서 혼자 하게 함 | 같이 키움 |
| 기술 허들 높음 | 기술 허들 없음 |
| 유지보수 본인 책임 | 본사가 관리 |
| 실패 확률 높음 | 성공 확률 높음 |

**실제 테스트 결과:** 비개발자에게 턴키로 전달 시도 → 실패 → Collaborator 모델로 피봇

### Exit 전략

멤버가 독립하고 싶을 때:
1. 누적 수익의 30%를 양도 비용으로 지불
2. 모든 레포를 Fork/복사
3. 독립 개발 가이드 제공 (Claude Code + PWA/APK)

## 5. AI 시대 적합성

### 패러다임 변화

**과거:**
```
회사 = 정규직 팀 + 사무실 + 협업 도구 + 고정비
```

**현재 (AI 시대):**
```
1인 + Claude = 풀스택 개발
필요시 용역 계약
고정비 최소화
```

### KOOSY가 이 패러다임에 맞는 이유

1. **GitHub 중심** - 협업 도구 불필요
2. **Claude Code 통합** - AI가 개발 파트너
3. **정적 사이트** - 서버 관리 불필요
4. **자동화 파이프라인** - GitHub Actions
